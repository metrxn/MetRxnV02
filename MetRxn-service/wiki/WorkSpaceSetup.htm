1.           Setting up workspace

Install Java (Skip this step if Java has been installed)

·         Install the latest version of JDK from http://www.oracle.com/technetwork/java/javase/downloads/index.html

·         Setting up java home path variable :

o   Right click on My Computer

o   Click on properties

o   Click on the Advanced System Settings link on the left hand side menu

o   Click on the Advanced tab and press the Environment variables

o   Under system variables find the variable called “PATH”

o   Click on edit.

o   Move to the end of the line in the variable value.

o   Add a semicolon and add the path of the installed JDK to it (EX : C:\Program Files\Java\jdk1.6.0\bin)

o   Click apply and ok

Test if java is installed properly

·         Open command prompt and type javac it should give a output like :

Install maven

·         Download maven(binary zip)  from http://maven.apache.org/download.cgi

·         Extract the zip to any folder (say D:\maven)

·         Add a system variable in the environment variables list called MAVEN_HOME and set its value to the installed location (say D:\Maven)

·         Edit path variable and add %MAVEN_HOME%/bin; to it.

·         Click apply and save.

·         Open a cmd prompt and type mvn –version, you should get something like

Setting up git

·         Download and install git from http://git-scm.com/downloads

·         Click on start -> All programs -> Git -> Git bash

·         A window similar to command prompt opens up

·         Type git it should prompt with something like this:

$ git

usage: git [--version] [--help] [-c name=value]

           [--exec-path[=<path>]] [--html-path] [--man-path] [--info-path]

           [-p|--paginate|--no-pager] [--no-replace-objects] [--bare]

           [--git-dir=<path>] [--work-tree=<path>] [--namespace=<name>]

           <command> [<args>]

 

The most commonly used git commands are:

   add        Add file contents to the index

   bisect     Find by binary search the change that introduced a bug

   branch     List, create, or delete branches

   checkout   Checkout a branch or paths to the working tree

   clone      Clone a repository into a new directory

   commit     Record changes to the repository

   diff       Show changes between commits, commit and working tree, etc

   fetch      Download objects and refs from another repository

   grep       Print lines matching a pattern

   init       Create an empty Git repository or reinitialize an existing one

   log        Show commit logs

   merge      Join two or more development histories together

   mv         Move or rename a file, a directory, or a symlink

   pull       Fetch from and merge with another repository or a local branch

   push       Update remote refs along with associated objects

   rebase     Forward-port local commits to the updated upstream head

   reset      Reset current HEAD to the specified state

   rm         Remove files from the working tree and from the index

   show       Show various types of objects

   status     Show the working tree status

   tag        Create, list, delete or verify a tag object signed with GPG

Download Eclipse

·         Go to http://www.eclipse.org/downloads/ and download “Eclipse IDE for Java EE Developers”

·         Unzip the downloaded zip file.

·         Open the unzipped folder. Click on “eclipse” icon.

·         Eclipse will start up.

·         Integrating maven with eclipse

o   Click on help -> Eclipse market place

o   Search for maven Integration for Eclipse and click on install

·         Installing git plugin to eclipse :

o   Click on help -> Eclipse market place

o   Search for eGit

o   Click install

Setting up Apache Tomcat:

·         Download the zip from http://tomcat.apache.org/download-70.cgi

·         Unzip the contents to the folder where you would like to place your Tomcat.

·         Once unzipped go to apache-tomcat-7.0.42\bin and click on startup.bat.

·         Open a web browser and hit http://localhost:8080/. You should be able to view the tomcat’s home page.

 

 

 

 

 

 

 

 

2. Codebase

Code setup

·         To check out the code from git repository, open git bash and execute the following command

o   git clone https://github.com/ambikab/MetRxn.git

·         You should get a folder called MEtRxn created in your local (will be created on the same location from where you issued the clone command in step 1)

·         Navigate into MetRxn\MetRxn-service

·         Execute the following commands

o   mvn eclipse:eclipse

o   mvn clean install –DskipTests=true

·         Open eclipse and click on file -> Import -> Maven -> Existing Maven Projects -> Browse -> Open <location>\ MetRxn\MetRxn-service

·         Click ok. Then click finish.

Deploying application

·         After you execute the last command you should be able to see a message saying “Build success” on your screen.

·         Now go to <location>\ MetRxn\MetRxn-service\target

·         Copy the .war file from that location and paste it in <location>\ apache-tomcat-7.0.42\webapps

·         Start tomcat.

·         Hit http://localhost:8080/MetRxn-service/. You should be able to access the MEtRxn home page.
